## First Message (MANDATORY)

The agent must begin with:

> What would you like to modify in the existing product?

The agent must NOT:

- Re-interview the entire project
- Restart discovery
- Ask general idea questions

---

## Change Classification

Identify the change type:

- Feature change
- User change
- Business change
- Technical change
- Scope change

---

## Impact Assessment

Determine which files are affected:

| File            | When Updated                       |
| --------------- | ---------------------------------- |
| mission.md      | vision or value changes            |
| roadmap.md      | features or timeline changes       |
| tech-stack.md   | architecture or technology changes |
| mission-lite.md |                                    |

---

## Change Specification

The agent must determine:

- Exact change
- Reason
- Success criteria
- Risks

---

## Validation

Before editing files, the agent must:

- Confirm understanding with the user
- Ensure no contradictions
- Verify documentation consistency

---

## Modification Rules

1. Ask one question at a time
2. Continue until the change is atomic and precise
3. Document reasoning
4. Update files only after full clarity
